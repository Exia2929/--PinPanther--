# PinPanther 🐾 — 多螢幕視窗自動定位

**中文** ｜ [English](README.en.md)

> 像粉紅豹一樣，優雅又精準地把你的視窗「pin」到指定螢幕。

每次開啟某個程式時，自動把它的視窗丟到你指定的螢幕，並以「視窗 / 最大化 / 全螢幕(無邊框)」開啟。

**運作機制**：用圖形化介面把規則設定好 → 一鍵**產生一支 AutoHotkey v2 腳本** → 自動部署到開機啟動資料夾。
之後開機時，這支腳本常駐在背景，用 Windows 的「視窗建立」事件即時偵測，符合規則的視窗一出現就立刻定位。
GUI 本身只是**設定編輯器 / 腳本產生器 / 測試工具**，平常不需要常駐。

## 特色
- **開機定位交給 AHK 腳本**：由視窗建立事件驅動（比輪詢靈敏），外加「開機初掃 + 補掃 + 重套幾次」抓比腳本早出現、或開窗後會自己跳位的程式。
- **圖形化編輯器**：新增 / 編輯 / 刪除規則，下拉挑選目前有視窗的程式與螢幕。
- **一鍵擷取目前布局**：把每個螢幕上最上層的視窗一次擷取成多條規則——擺好版面後一鍵保存（自動偵測最大化／全螢幕／視窗，並在同程式跨多螢幕時用標題區分）。
- 每條規則可分別設定：執行檔、（選填）標題關鍵字、目標螢幕、開啟模式、是否啟用。
- **即時預覽**：在 GUI 裡先試套規則，確認沒問題再產生腳本部署（不必反覆重開機試錯）。
- **視窗模式記住自訂大小/位置**：把程式擺到喜歡的大小位置後一鍵擷取，腳本會自動還原。
- **螢幕編號一致**：GUI 與產生的腳本都用「左→右、上→下」排序，選的螢幕＝實體同一台，解析度變動也不跑掉。
- **中英文介面**：右上角 🌐 下拉即可即時切換中文 / English，偏好會記在設定檔；產生的 AHK 腳本系統列提示也跟著語言走。

## ⚠️ 安全與權限說明 (Security & Permissions)
- **管理員權限**：若目標程式（例如部分遊戲或系統工具）是以「系統管理員」身分執行，本程式也必須以管理員身分執行才能成功控制其視窗。
- **AHK 腳本安全**：本專案產生的 `PinPanther.ahk` 原始碼是公開透明的，你可以隨時檢查其內容。在以管理員權限執行任何腳本前，建議先行閱讀內容以確保安全。

## 系統需求
- Windows 10 / 11
- Python 3.8 以上（你目前是 3.14，沒問題）—— 只用標準函式庫，免裝任何套件。
- **AutoHotkey v2**（開機腳本要用它執行）。可直接在程式裡按「安裝 AutoHotkey v2」一鍵安裝（透過 winget），或自行到 <https://www.autohotkey.com/> 安裝；GUI 會自動偵測安裝路徑。

## 執行方式
- 雙擊 **`啟動 MonitorPin.bat`**（用 `pythonw` 啟動，不會跳黑色主控台視窗），或
- 在此資料夾執行：`python monitor_pin.py`

## 使用步驟
介面由上而下分成三個步驟，照順序操作即可：

1. **① 設定規則**：按 **＋ 新增** 建立規則
   - **執行檔名稱**：例如 `discord.exe`、`vivaldi.exe`（下拉選單會列出目前有視窗的程式）。
   - **視窗標題包含**（選填）：同一程式想依標題分流時才填。
   - **開啟在哪個螢幕**、**開啟模式**（視窗 / 最大化 / 全螢幕(無邊框)）。
   - 懶人法：先把各螢幕的視窗擺成你要的版面，按 **📸 一鍵擷取目前布局**，一次把每個螢幕最上層的視窗存成規則（可選擇「取代全部／加入現有」）。
2. **② 測試規則**（不影響開機）：按 **▶ 開始預覽監控** 後開那些程式，或對已開著的視窗按 **套用到目前開啟的視窗**，確認會不會跑到定位。
3. **③ 部署到開機**：
   - 在新電腦上若還沒裝，先按 **安裝 AutoHotkey v2**（用 winget 一鍵安裝；裝過會自動偵測，不必再按）。
   - 按 **✦ 產生並部署到開機啟動** —— 產生 `PinPanther.ahk` 並在開機啟動資料夾建立捷徑。
   - 想立刻看效果按 **立即測試執行**；改了規則後重按「產生並部署」即覆蓋。

> 各程式「自己的開機自啟動」維持原樣即可——腳本是在它們的視窗出現時才接手定位，不是由腳本去啟動它們。

## 開啟模式說明
- **視窗**：預設在目標螢幕工作區置中、大小約為工作區的 70%；若有設定自訂大小則用自訂值（見下）。
- **最大化**：移到目標螢幕後最大化（保留標題列，工作列仍在）。
- **全螢幕(無邊框)**：移除邊框與標題列，覆蓋整個螢幕。注意這是「無邊框視窗化全螢幕」，
  不是遊戲的「獨佔全螢幕」（獨佔全螢幕由程式本身控制，外部無法強制）。

## 視窗模式記住自訂大小／位置
1. 把目標程式手動拖到你想要的螢幕、調成喜歡的大小與位置。
2. 在規則清單選取該規則，按 **擷取目前視窗大小/位置** —— 會把目前的寬高與
   「相對該螢幕工作區左上角」的座標存進規則。
3. 確認該規則的模式為 **視窗**。之後每次開啟，就會還原成這個大小與位置。

也可在「新增/編輯規則」視窗裡手動填寫 寬 / 高 / 相對X / 相對Y；四個欄位留空＝自動置中 70%。

## 開機腳本（AutoHotkey）
- **安裝 AutoHotkey v2**：陌生／全新環境一鍵安裝（透過 winget）；偵測到已安裝會顯示為「重新安裝」。沒有 winget 時會改開官方下載頁。
- **產生並部署到開機啟動**：產生 `PinPanther.ahk`（存在專案資料夾，可版控/直接看），
  並在 `shell:startup` 建立一個指向 `AutoHotkey64.exe` 的捷徑，開機時自動跑腳本。
- **移除開機啟動**：刪掉該捷徑（腳本檔本身保留）。
- **立即測試執行**：用最新規則馬上啟動腳本（不必重開機）。腳本常駐系統列，右鍵圖示 → 結束。
- **開啟腳本資料夾**：在檔案總管中定位到 `PinPanther.ahk`。
- 腳本是「**反應式**」的：靠視窗事件即時搬移。提醒：遊戲若用**獨佔全螢幕**，外部仍無法搬動，
  請先在遊戲內改成「無邊框視窗」。

## 設定檔
- 規則會自動存到同目錄的 `monitor_pin_rules.json`，下次開啟自動載入。
- `PinPanther.ahk` 是依規則產生的成品；改規則後重按「產生並部署」即覆蓋，不需手動編輯。

## 常見問題
- **某些程式搬不動？** 若該程式以「系統管理員」身分執行，AHK 腳本也要有同等權限才能搬它的視窗
  （可把開機捷徑設為以系統管理員執行，或在遊戲/程式端降權）。GUI 預覽測試時同理。
- **找不到 AutoHotkey？** 按鈕會提示，請先安裝 AutoHotkey **v2**；GUI 會自動在常見路徑與登錄檔尋找。
- **遊戲全螢幕無效？** 多數遊戲使用獨佔全螢幕，請在遊戲內改為「無邊框視窗」後，再用「全螢幕(無邊框)」定位。
- **移錯螢幕？** 按「重新偵測螢幕」，並確認規則裡選的螢幕編號正確（GUI 與腳本編號一致）。
