# PinPanther 🐾 — Multi-Monitor Window Auto-Positioning

[中文](README.md) ｜ **English**

> Like the Pink Panther — pin your windows to the right monitor, gracefully and precisely.

Every time you open an app, PinPanther sends its window to the monitor you chose and opens it in **Windowed / Maximized / Fullscreen (borderless)** mode.

**How it works**: configure your rules in the GUI → with one click, **generate an AutoHotkey v2 script** → deploy it to the startup folder automatically.
After that, at boot the script sits in the background and uses Windows' "window created" event to detect matching windows the moment they appear, then positions them.
The GUI itself is only a **settings editor / script generator / test tool** — it does not need to run all the time.

## Features
- **Startup positioning is handled by the AHK script**: driven by window-created events (snappier than polling), plus an "initial sweep + periodic re-sweep + re-apply a few times" to catch apps that appear before the script or that move themselves after opening.
- **Graphical editor**: add / edit / delete rules, with drop-downs to pick currently-windowed apps and monitors.
- **One-click layout capture**: capture the top-most window on every monitor into multiple rules at once — arrange your layout, then save it in one click (auto-detects maximized / fullscreen / windowed, and uses the title to distinguish the same app spread across monitors).
- Each rule can be set individually: executable, (optional) title keyword, target monitor, open mode, and whether it's enabled.
- **Live preview**: try rules inside the GUI first and confirm they work before generating and deploying the script (no need to reboot repeatedly to test).
- **Windowed mode remembers custom size/position**: arrange an app at the size/position you like, capture it in one click, and the script restores it.
- **Consistent monitor numbering**: both the GUI and the generated script sort monitors "left→right, top→bottom", so the monitor you pick is the same physical screen even if resolutions change.
- **Bilingual UI**: switch between 中文 / English instantly from the 🌐 drop-down at the top-right; your choice is saved to the config file, and the generated AHK script's tray hints follow the language too.

## ⚠️ Security & Permissions
- **Administrator Privileges**: If a target application (e.g., some games or system tools) is running as **Administrator**, this program must also be run as Administrator to successfully control its windows.
- **AHK Script Safety**: The `PinPanther.ahk` script generated by this project is transparent and open-source. You can inspect its content at any time. It is always recommended to review the content of any script before running it with elevated privileges.

## Requirements
- Windows 10 / 11
- Python 3.8+ (you're on 3.14, which is fine) — uses only the standard library, no packages to install.
- **AutoHotkey v2** (needed to run the startup script). You can install it in one click via the in-app "Install AutoHotkey v2" button (uses winget), or install it yourself from <https://www.autohotkey.com/>; the GUI auto-detects the install path.

## Running it
- Double-click **`啟動 MonitorPin.bat`** (launches via `pythonw`, so no black console window pops up), or
- Run in this folder: `python monitor_pin.py`

## Steps
The UI is laid out top-to-bottom in three steps — just follow them in order:

1. **① Configure rules**: click **＋ Add** to create a rule
   - **Executable name**: e.g. `discord.exe`, `vivaldi.exe` (the drop-down lists apps that currently have windows).
   - **Window title contains** (optional): only fill this in when you want to route the same app by title.
   - **Open on which monitor**, **Open mode** (Windowed / Maximized / Fullscreen (borderless)).
   - Lazy shortcut: arrange each monitor's window the way you want, then click **📸 Capture current layout** to save the top-most window on each monitor as rules at once (you can choose "replace all / add to existing").
2. **② Test rules** (no effect on startup): click **▶ Start preview monitoring** and then open those apps, or click **Apply to currently open windows** for windows that are already open, to confirm they get positioned.
3. **③ Deploy to startup**:
   - On a new PC where it isn't installed yet, click **Install AutoHotkey v2** first (one-click install via winget; if already installed it's auto-detected and you can skip this).
   - Click **✦ Generate & deploy to startup** — generates `PinPanther.ahk` and creates a shortcut in the startup folder.
   - To see the effect immediately, click **Test-run now**; after changing rules, click "Generate & deploy" again to overwrite.

> Leave each app's own "start at login" as-is — the script takes over positioning when their windows appear; it does not launch them.

## Open modes
- **Windowed**: by default centered in the target monitor's work area at about 70% of its size; if a custom size is set, that value is used (see below).
- **Maximized**: moved to the target monitor and then maximized (keeps the title bar; the taskbar stays).
- **Fullscreen (borderless)**: removes the border and title bar and covers the whole monitor. Note this is "borderless windowed fullscreen", not a game's "exclusive fullscreen" (exclusive fullscreen is controlled by the app itself and can't be forced from outside).

## Windowed mode: remembering custom size/position
1. Manually drag the target app to the monitor you want and resize it to your liking.
2. Select that rule in the list and click **Capture selected rule's size/position** — this stores the current width/height and the coordinates "relative to that monitor's work-area top-left" into the rule.
3. Make sure the rule's mode is **Windowed**. From then on, every time it opens it'll be restored to that size and position.

You can also fill in W / H / Rel X / Rel Y by hand in the "Add/Edit rule" window; leaving all four blank = auto-center at 70%.

## Startup script (AutoHotkey)
- **Install AutoHotkey v2**: one-click install for an unfamiliar / brand-new environment (via winget); if it's already installed the button shows "Reinstall". If winget isn't available, it opens the official download page instead.
- **Generate & deploy to startup**: generates `PinPanther.ahk` (kept in the project folder, version-controllable / readable) and creates a shortcut in `shell:startup` pointing at `AutoHotkey64.exe`, so the script runs automatically at boot.
- **Remove from startup**: deletes that shortcut (the script file itself is kept).
- **Test-run now**: launches the script with the latest rules immediately (no reboot needed). The script runs in the tray; right-click the icon → quit.
- **Open script folder**: locates `PinPanther.ahk` in File Explorer.
- The script is **reactive**: it moves windows in real time via window events. Reminder: if a game uses **exclusive fullscreen**, it still can't be moved from outside — switch it to "borderless windowed" inside the game first.

## Config files
- Rules are saved automatically to `monitor_pin_rules.json` in the same folder and loaded on next launch. The chosen UI language is stored there too.
- `PinPanther.ahk` is the product generated from your rules; after changing rules, click "Generate & deploy" again to overwrite it — no manual editing needed.

## FAQ
- **Some apps won't move?** If the app runs as **Administrator**, the AHK script needs equivalent privileges to move its window (set the startup shortcut to run as administrator, or lower the app/game's privileges). The same applies to the GUI preview test.
- **Can't find AutoHotkey?** The button will prompt you; install AutoHotkey **v2** first. The GUI auto-searches common paths and the registry.
- **Game fullscreen has no effect?** Most games use exclusive fullscreen — switch to "borderless windowed" inside the game, then position it with "Fullscreen (borderless)".
- **Moved to the wrong monitor?** Click "Re-detect monitors" and make sure the monitor number in the rule is correct (GUI and script numbering match).
